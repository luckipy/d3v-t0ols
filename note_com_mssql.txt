xp_cmdshell là một extended stored procedure trong Microsoft SQL Server cho phép bạn chạy lệnh hệ điều hành (cmd) trực tiếp từ bên trong SQL Server.

Cách xp_cmdshell hoạt động:
- SQL Server nhận lệnh EXEC xp_cmdshell
- Nó gọi xuống Windows API
- Tạo một process giống như mở cmd.exe
- Thực thi command bạn truyền vào
- Thu output (stdout) và trả về dưới dạng result set


Nó chạy dưới quyền của SQL Server service account.


Stored Procedure là một tập hợp các câu lệnh SQL được lưu sẵn trong database và có thể gọi lại nhiều lần. Nó được viết bằng SQL.
Ví dụ:
CREATE PROCEDURE GetUsers
AS
BEGIN
    SELECT * FROM Users;
END;

Sau đó chỉ cần gọi: EXEC GetUsers;

Extended stored procdures là các DLL được viết bằng ngôn ngữ lập trình, ví dụ như C

Những điều kiện bắt buộc để có thể remote access vào MSSQLSerrver:
- SQL Server service đang chạy
- TCP/IP protocol được enable
- Port 1433 (hoặc tùy chỉnh trong Configuration Manager) đang mở và firewall cho phép access
- Remote Connections được bật
- Authentication Mode: Cho phép cả 2 mode login: Windows Authentication và SQL Server authentication


Khi sử dụng Windows Authentication nghĩa là nó tự lấy username hiện tại (local hoặc domain) để thực hiện xác thực, nó có thể sử dụng NTLM/Kerberos để xác thực, password sẽ không được lưu trong SQL Server.
Khi sử dụng SQL Server Authentication thì có thể sử dụng username/password trong SQL để xác thực.

Điều đặc biệt là có thể sử dụng tài khoản của Windows để remote login (phải là username/password của account windows chứ không phải của SQL). Có thể truyền vào (<hostname/domain>/username hoặc username)
Nó chỉ có thể remote login với chính Account cài đặt service thôi, các account khác không được. (hoặc có thể phải cấu hình thêm ở đâu đó)
Đặc biệt, khi chỉ sử dụng Windows Authentication mà không tick chọn Mixed mode thì không thể đăng nhập bằng các account (username/password) của SQL (cả local và remote), nhưng khi remote vẫn đăng nhập với tài khoản windows cài đặt service.
